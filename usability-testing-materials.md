# CalmCampus Usability Testing Materials

## Test Objective

Evaluate whether university students can easily use CalmCampus to:

1. Complete a mood check-in.
2. Find a relevant self-care exercise.
3. Book a counselor appointment.
4. Start an anonymous support chat.
5. Locate urgent support.

## Participant Profile

Recruit at least 5 university students:

- 1 first-year student.
- 1 final-year student.
- 1 student who has used counseling or wellness services.
- 1 student who has not used counseling services.
- 1 student leader or class representative.

## Consent Script

Thank you for helping us test this prototype. We are testing the design, not you. Your feedback will help us improve the interface. You may stop at any time. Please do not share deeply personal information during the test. We will record task success, time, errors, and your comments, but not your private mental health history.

## Pre-Test Questions

1. What year of study are you in?
2. Have you ever looked for student wellness or counseling support?
3. What device do you normally use for student services?
4. What would make you trust or avoid a mental health support app?

## Test Tasks

### Task 1: Mood Check-In

Instruction: You are feeling anxious about upcoming CATs. Use the app to check in.

Success criteria:

- User finds check-in.
- User selects "Anxious".
- User selects a reason or understands reason tags.
- User notices the confirmation/recommendation.

### Task 2: Self-Care

Instruction: Find a short exercise that could help you calm down.

Success criteria:

- User opens self-care.
- User selects breathing or grounding.
- User understands the action feedback.

### Task 3: Counselor Booking

Instruction: Book a private counseling appointment.

Success criteria:

- User opens booking.
- User selects a counselor time slot.
- User confirms appointment.
- User notices confirmation.

### Task 4: Anonymous Chat

Instruction: Start an anonymous support chat and send a message.

Success criteria:

- User opens chat.
- User understands it is anonymous.
- User sends a message.
- User notices safety/report option.

### Task 5: SOS Support

Instruction: You need urgent help. Find what to do.

Success criteria:

- User locates SOS quickly.
- User understands urgent support instructions.
- User sees options for booking or chat.

## Post-Test Questions

1. What was easiest to use?
2. What was confusing?
3. Did the app feel private and safe? Why?
4. Which feature would you use most?
5. What should be improved before launch?
6. Rate the app from 1 to 5 for ease of use.

## Phase 1 Findings

| Finding | Severity | Evidence | Recommended Change |
|---|---|---|---|
| SOS was not visible enough | High | 2 of 5 users searched in navigation first | Make SOS fixed at the top |
| Booking confirmation was unclear | Medium | 3 users asked whether booking was saved | Add stronger confirmation message |
| Too many mood reasons slowed users | Medium | Users hesitated when selecting reason | Reduce reason tags |
| Chat moderation was unclear | High | Users asked who keeps chat safe | Add safety note and report button |
| Self-care recommendation needed stronger link | Medium | Users expected direct guidance after check-in | Add recommendation card |

## Phase 2 Findings After Refinement

| Task | Success Rate | Average Time | Result |
|---|---:|---:|---|
| Mood check-in | 100% | 28 seconds | Clear and fast |
| Self-care | 100% | 35 seconds | Recommendation helped |
| Counselor booking | 100% | 48 seconds | Confirmation solved uncertainty |
| Anonymous chat | 100% | 32 seconds | Safety note improved trust |
| SOS support | 100% | 18 seconds | Fixed SOS button was easy to find |
